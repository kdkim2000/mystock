## 1. 시스템 목적 및 추진배경

my-stock은 개인 주식 투자자가 Google Sheets에 기록한 매매 내역을 바탕으로 포트폴리오 수익률과 리스크를 실시간으로 파악하고, KIS Open API·DART API·OpenAI를 연동하여 종목별 재무 정보와 AI 매매 가이드를 한 화면에서 제공함으로써 데이터 기반 투자 의사결정을 지원하는 단일 사용자용 웹 애플리케이션이다.

기존 개인 투자자는 증권사 앱(시세), DART 웹사이트(공시·재무), ChatGPT(AI 분석), Google Sheets(매매 일지)를 따로 열어 수동으로 데이터를 취합해야 하는 불편함이 있었다. my-stock은 이 분산된 정보를 하나의 대시보드로 통합하여, 별도 데이터베이스 없이 Google Sheets를 데이터 저장소 겸 캐시로 활용하는 경량 구조로 설계되었다.

### 핵심 가치 제안

| 가치 | 설명 |
|:---|:---|
| 통합 대시보드 | 포트폴리오 KPI·차트·종목 분석·매매 내역을 한 화면에서 조회 |
| 종목 상세 통합 | 시세·가치평가·재무·DART 공시·AI 분석을 종목 상세 한 페이지에서 제공 |
| AI 매매 가이드 | 과거 매매 일지와 현재 종목 데이터를 결합한 gpt-4o-mini 맞춤형 분석 |
| 경량 인프라 | Google Sheets = DB + 캐시, 별도 DB 서버 불필요, Vercel 서버리스 배포 |

## 7. 데이터 모델 (Google Sheets)

Google Sheets가 데이터베이스 역할을 수행한다. 단일 스프레드시트 내 5개 탭으로 구성된다.

### 7-1. 매매내역 탭 (`GOOGLE_SHEET_NAME`, 기본: `매매내역`)

TypeScript 타입: `SheetTransactionRow`

| 컬럼 | 타입 | 설명 |
|:---|:---:|:---|
| Date | string | 거래 날짜 (YYYY-MM-DD) |
| Ticker | string | 한국 종목명 |
| Type | string | 매수 / 매도 |
| Quantity | number | 거래 수량 (주) |
| Price | number | 거래 단가 (원화) |
| Journal | string | 매매 메모 (매매 일지) |
| Tags | string | 전략 태그 (쉼표 구분) |

### 7-2. KIS/DART 캐시 탭 (`_TICKER_CACHE_`)

| 속성 | 내용 |
|:---|:---|
| 키 구조 | `{code}_{section}` (예: `005930_price`, `005930_financial`) |
| TTL | 30분 (KIS), 1시간 (DART 재무) |
| 용도 | API 응답 JSON 직렬화 저장, 서버리스 인스턴스 간 공유 |

### 7-4. 종목코드 마스터 (`GOOGLE_SHEET_TICKER_MASTER`, 선택)

| 컬럼 | 설명 |
|:---|:---|
| Ticker | 종목명 (한국어) |

## 8. 3계층 캐싱 전략

서버리스 환경의 인스턴스 무상태 특성을 보완하기 위한 3단계 캐시 계층.

```
요청 도달
    │
    ▼ 1계층: globalThis (인메모리)
    ├─ HIT → 즉시 반환 (< 1ms)
    └─ MISS
         │
         ▼ 2계층: /tmp/ (파일시스템)
         ├─ HIT → 반환 후 globalThis에 복사
         └─ MISS
              │
              ▼ 3계층: Google Sheets (_TICKER_CACHE_ / _AI_CACHE_)
              ├─ HIT → 반환 후 상위 계층에 복사
              └─ MISS → 외부 API 호출 → Sheets에 저장 → 반환
```

### 계층별 특성

| 계층 | 저장소 | TTL | 특성 |
|:---:|:---|:---:|:---|
| 1계층 | `globalThis` | 인스턴스 생존 시간 | KIS 액세스 토큰, Google OAuth 토큰 |
| 2계층 | `/tmp/` (Vercel) / `.next/cache/` (로컬) | 24시간 | KIS 토큰 파일 캐시 |
| 3계층 | Google Sheets `_TICKER_CACHE_` | 30분 (KIS) / 1시간 (DART) | 전체 인스턴스 공유, 영구 저장 |
| 3계층 | Google Sheets `_AI_CACHE_` | 7일 | OpenAI 분석 결과, 강제 갱신 지원 |

## 9. 외부 시스템 연동

| 외부 시스템 | 연동 목적 | 인증 방식 | 주요 제약사항 |

| **Google OAuth 2.0** | 사용자 인증 및 세션 관리 | OAuth 2.0 Client ID/Secret | ALLOWED_EMAIL로 단일 사용자 제한 |
