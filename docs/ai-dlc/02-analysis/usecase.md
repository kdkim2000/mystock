## 3. 유즈케이스 상세

### UC-001: Google 로그인

| 항목 | 내용 |
|:---|:---|
| 액터 | 투자자 (주요), Google OAuth (외부) |
| 사전 조건 | 사용자가 `/auth/signin` 페이지에 접근한 상태 |
| 사후 조건 | JWT 세션이 발급되어 대시보드(`/dashboard`)로 리다이렉트 |
| 연관 FR | FR-001 |

**기본 흐름 (주요 성공 시나리오)**

| 단계 | 액터/시스템 | 처리 내용 |
|:---:|:---:|:---|
| 1 | 투자자 | "Google로 로그인" 버튼 클릭 |
| 2 | 시스템 | Google OAuth 2.0 인증 화면으로 리다이렉트 |
| 3 | Google OAuth | 사용자 자격증명 확인 후 인증 코드 반환 |
| 4 | 시스템 | 인증 코드로 세션 생성. ALLOWED_EMAIL 설정 시 이메일 일치 여부 검증 |
| 5 | 시스템 | JWT 세션 발급 후 `/dashboard`로 리다이렉트 |

**대안 흐름**

- 이미 유효 세션 보유 시: `/auth/signin` 접근 즉시 `/dashboard`로 리다이렉트

**예외 흐름**

| 예외 조건 | 처리 내용 |
|:---|:---|
| ALLOWED_EMAIL 불일치 | HTTP 401 반환, 로그인 실패 메시지 표시 |
| Google OAuth 인증 실패 | 인증 오류 메시지 표시, 재시도 유도 |
