## 1. 사업개요

### 1) 사업목적

개인 주식 투자자가 Google Sheets에 기록한 매매 내역을 바탕으로 포트폴리오 수익률·리스크를 실시간으로 파악하고, KIS·DART·OpenAI 연동을 통해 종목별 재무 정보와 AI 매매 가이드를 한 화면에서 제공함으로써 데이터 기반 투자 의사결정을 지원한다.

### 2) 사업개요

- **프로젝트명**: my-stock
- **플랫폼**: 웹 애플리케이션 (Next.js 15 App Router, Vercel 배포)
- **주요 사용자**: 단일 개인 투자자 (Google OAuth 이메일 허용 목록으로 접근 제한)
- **핵심 데이터 소스**: Google Sheets(매매 기록), KIS Open API(실시간 시세·재무), DART(공시·재무제표), OpenAI(AI 분석)

### 3) 추진 배경

개인 투자자는 여러 서비스(증권 앱, 전자공시, AI 도구)를 오가며 정보를 수집하는 불편함이 있다. 또한 자신의 매매 이력과 시장 데이터를 결합한 맞춤형 포트폴리오 분석 도구가 부재하다. my-stock은 이 갭을 메우기 위해 기존 Google Sheets 매매 일지를 데이터베이스 겸 캐시로 활용하여 별도 DB 없이 운영 가능한 개인용 투자 대시보드를 구현한다.

### 4) 사업 내용

1. **대시보드** — 실현손익·평가손익·승률·총자산 요약, 포지션 집중도·손익 기여도 차트, 누적 수익금 시계열, 종목별 분석 테이블, 전략 태그별 성과, 매매 내역 테이블
2. **종목 상세 페이지** — 13개 섹션(시세·재무·비율·추정실적·매매동향·투자의견·DART·포트폴리오·보조지표·AI 분석·매매 일지 등)
3. **데이터 갱신** — 종목별 통합 갱신 버튼 (Vercel 서버리스 1개 함수에서 KIS/DART 병렬 호출 → Google Sheets 캐시 저장)
4. **AI 분석** — 종목 정보 + 최근 매매 일지를 컨텍스트로 OpenAI gpt-4o-mini 호출, 결과 캐시(7일)

### 2) 인프라 구성

```
[브라우저]
  └─ HTTPS → [Vercel Edge]
       ├─ Next.js App Router (SSR/CSR)
       ├─ API Route Handlers (서버리스)
       │    ├─ /api/sheets/*      → Google Sheets API v4
       │    ├─ /api/kis/*         → KIS Open API
       │    ├─ /api/dart/*        → DART 전자공시 API

       └─ 캐시 계층
            ├─ globalThis (인메모리, 함수 인스턴스 내)
            ├─ /tmp/ 파일 (KIS 토큰, Vercel)
```
