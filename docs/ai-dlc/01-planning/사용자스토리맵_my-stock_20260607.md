| When | AI 분석 섹션이 렌더링되거나 "AI 갱신" 버튼을 클릭하면 |
| Then | 캐시 HIT 시 저장된 마크다운 분석 결과가 즉시 표시되고, 강제 갱신 시 OpenAI API를 호출하여 새 분석 결과를 `_AI_CACHE_`에 저장 후 표시하며, 생성 날짜가 함께 표시된다 |

| 항목 | 내용 |

**스토리**

**수락 기준**

| 단계 | 내용 |

| When | 매매 일지 섹션이 렌더링되면 |
| Then | 해당 종목 거래를 날짜 내림차순으로 날짜·유형(매수/매도)·수량·단가·메모(Journal) 컬럼의 테이블로 표시하며, 메모가 없는 행은 "-"로 표시된다 |

### US-022 — 데이터 통합 갱신 (갱신 버튼)

| 항목 | 내용 |
|:---|:---|
| US-ID | US-022 |
| 제목 | 갱신 버튼 한 번으로 KIS·DART 전체 캐시를 최신화 |
| Activity | 데이터 갱신 |
| Task | 통합 갱신 버튼 실행 |
| 관련 FR | FR-022 |
| 우선순위 | Must Have |
| 페르소나 목표 | G-1/G-2/G-3 공통 전제 (데이터 최신성) |

**스토리**

> 김데이터로서, 최신 시장 데이터를 기반으로 의사결정하기 위해 종목 상세 페이지의 갱신 버튼 하나로 KIS와 DART API를 병렬 호출하여 Google Sheets 캐시를 갱신하고, 갱신 완료 후 화면에 최신 데이터를 반영하고 싶다.

**수락 기준**

| 단계 | 내용 |
|:---|:---|
| Given | 종목 상세 페이지에서 갱신 버튼이 활성화된 상태에서 |
| When | 갱신 버튼을 클릭하면 |
| Then | `POST /api/ticker/[code]/refresh`가 호출되어 KIS·DART API를 병렬 요청하고, 60초 이내에 `_TICKER_CACHE_` 탭이 갱신된 후, 성공 토스트 메시지와 함께 모든 섹션의 데이터가 최신값으로 반영되며, 진행 중에는 버튼이 비활성화(스피너) 상태를 표시한다 |
