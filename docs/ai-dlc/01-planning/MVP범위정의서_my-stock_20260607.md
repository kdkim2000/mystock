## 1. MoSCoW 분류표

### 분류 기준

| 분류 | 기준 |
|:---|:---|
| Must Have | 없으면 서비스 핵심 목적(포트폴리오 통합 대시보드) 자체가 불가능 |
| Should Have | 목적 달성에 중요하나 다른 방법으로 임시 우회 가능 |
| Could Have | 있으면 경험 향상, 없어도 핵심 운영 가능 |
| Won't Have (v1) | 이번 버전 명시 제외, 후속 버전 이연 |

## 5. 비기능 요구사항 v1.0 포함 범위

비기능 요구사항은 서비스 운영 전제 조건으로 전체 v1.0에 포함한다.

### 보안 요구사항 (SR) — 전체 포함

| SR-ID | 명칭 | v1.0 | 구현 방법 |
|:---|:---|:---:|:---|
| SR-001 | Google OAuth 2.0 인증 | ✅ | NextAuth v4 + middleware.ts 전역 적용 |
| SR-002 | 이메일 기반 접근 제한 | ✅ | ALLOWED_EMAIL 환경변수, 미들웨어 검사 |
| SR-003 | API 키 보호 | ✅ | .env.local / Vercel Env Vars, 클라이언트 노출 0건 |
| SR-004 | HTTPS 강제 | ✅ | Vercel 배포 기본 제공 |
| SR-005 | 세션 만료 처리 | ✅ | API 401 → UI 안내 (리다이렉트 없음) |

### 성능 요구사항 (PR) — 전체 포함

| PR-ID | 명칭 | 목표값 | 달성 방법 |
|:---|:---|:---|:---|
| PR-001 | 종목 상세 초기 로딩 | 3초 이내 (캐시 HIT) | _TICKER_CACHE_ 30분 TTL |
| PR-002 | 데이터 갱신 완료 | 60초 이내 | Vercel maxDuration=60, KIS 병렬 호출 |
| PR-003 | 대시보드 로딩 | 5초 이내 | Google Sheets 캐시 활용 |
| PR-004 | KIS Rate Limit | 400ms 이상 간격 | KIS_THROTTLE_MS=400, 미준수 시 EGW00133 |

### 품질 요구사항 (QR) — 기본 수준 포함

| QR-ID | 명칭 | v1.0 수준 | v1.1 고도화 |
|:---|:---|:---|:---|
| QR-001 | 다크/라이트 모드 | next-themes 기본 설정 | CSS 변수 완성도 향상 |
| QR-002 | 반응형 레이아웃 | PC 우선 + 모바일 기본 가독성 | 모바일 최적화 |

| QR-004 | 에러 상태 UI | API 실패·데이터 없음 fallback | 상세 에러 분류 |
