## 2. 페르소나 카드

---

### PS-001 · 김데이터

| 항목 | 내용 |
|:---|:---|
| 가상 이름 | 김데이터 |
| 나이 / 직군 | 40대 초반 / 대기업 재직 직장인 (엔지니어 계열) |
| 역할 | 직장인 개인 주식 투자자 |
| 기술 친숙도 | **높음** — Next.js·Vercel·Google Sheets API를 직접 구축할 수 있는 개발자 수준 |
| 투자 경력 | 국내 주식 투자 5년 이상, Google Sheets에 매매 내역 직접 기록 중 |
| 주요 관심 종목 | 한국 코스피·코스닥 상장 종목 (대형주 중심, 일부 성장주) |

#### 주요 목표 (Goal)

| # | 목표 |
|:---:|:---|
| G-1 | 증권 앱·DART·ChatGPT·Sheets를 따로 오가지 않고, **한 화면**에서 포트폴리오 전체 현황(실현·평가손익, 승률, 총자산)을 즉시 파악한다 |
| G-2 | 종목별 재무 지표(PER·PBR·ROE·부채비율 등)와 DART 공시를 **통합 조회**하여 매수·매도 의사결정에 직접 활용한다 |
| G-3 | 과거 매매 일지와 AI 분석을 결합해 **반복적 실수를 줄이고** 자신만의 투자 전략을 지속 개선한다 |

#### 불편한 점 (Pain)

| # | Pain |
|:---:|:---|
| P-1 | 증권 앱(시세), DART 웹사이트(공시·재무), ChatGPT(AI 분석), Google Sheets(매매 일지)를 따로 열어 **수동으로 데이터를 취합**해야 해서 시간이 많이 걸린다 |
| P-2 | 보유 종목의 평가손익·승률·포지션 집중도를 **직접 계산하거나 Sheets 수식으로 관리**하는 것이 번거롭고, 실수가 발생하기 쉽다 |
| P-3 | 과거 매매 이력과 현재 종목 상황을 **연결하여 분석**하는 일관된 도구가 없어, 같은 실수를 반복하는 경향이 있다 |

#### 원하는 것 (Gain)

| # | Gain |
|:---:|:---|
| Ga-1 | 대시보드 한 페이지에서 **포트폴리오 KPI(4개 카드) + 집중도·손익 차트 + 종목 분석 테이블**을 한눈에 확인 |
| Ga-2 | 종목 클릭 하나로 **시세·가치평가·재무·DART·AI 분석**을 순서대로 탐색할 수 있는 상세 페이지 |
| Ga-3 | **갱신 버튼 하나**로 KIS·DART 최신 데이터를 자동 업데이트하고, AI가 매매 일지 맥락에서 맞춤형 투자 조언 제공 |

#### 주요 사용 시나리오

**시나리오 1 — 일과 후 포트폴리오 점검 (가장 빈번)**

```
저녁 퇴근 후 대시보드 접속
→ 총자산·승률·실현손익 카드 확인
→ 포지션 집중도 차트에서 비중 쏠린 종목 확인
→ 종목별 분석 테이블에서 손익 기여도 상위 종목 클릭
→ 종목 상세 페이지 — 현재가·재무비율·AI 분석 확인
→ 추가 매수 or 매도 의사결정
```

**시나리오 2 — 데이터 갱신 및 AI 분석 요청 (주 1~2회)**

```
종목 상세 페이지 접속
→ 갱신 버튼 클릭 (KIS·DART 병렬 호출, 최대 60초)
→ 최신 가치평가·재무·투자의견 데이터 로드 완료
→ AI 분석 섹션 — 최근 매매 일지 기반 gpt-4o-mini 분석 결과 확인
→ 투자 가이드 참고 후 매매 일지에 메모 추가
```

#### 기술·환경 특성

| 항목 | 내용 |
|:---|:---|
| 접속 환경 | PC 브라우저 우선 (크롬), 간헐적 모바일 확인 |
| 사용 빈도 | 평일 저녁 1회 접속, 갱신은 주 1~2회 |
| 선호 UI | 정보 밀도 높은 테이블·차트 선호, 스켈레톤 로딩 허용 |
| 데이터 신뢰 | 캐시 HIT 기준 3초 이내 로딩 기대, 갱신 중 60초 대기 허용 |
| 다크 모드 | 저녁 사용이 많아 다크 모드 선호 |
