## Lessons Learned

- **단일 사용자 시스템의 페르소나**: 사용자가 1명이어도 페르소나 카드를 작성하면 이후 유스케이스·화면 설계 시 "누가 왜 이 기능을 쓰는가"를 명확히 할 수 있어 유용함.
- **기술 친숙도 "높음" 설정의 영향**: 에러 메시지·기술 용어를 그대로 노출해도 되고, 고급 기능(강제 갱신, 세션 만료 UI 안내 등)에 대한 추가 설명을 최소화할 수 있음 — 이를 QR 설계 시 반영할 것.
- **시나리오 2개로 충분**: 단일 사용자는 사용 패턴이 예측 가능하므로 2개 시나리오(일상 점검 + 데이터 갱신)가 FR-001~FR-022를 거의 커버함. 유스케이스 단계에서 세분화하면 됨.
- **액터-유스케이스 연계 힌트 포함**: 페르소나 파일 하단에 다음 스킬(`ai-dlc-usecase-create`) 입력용 유스케이스 후보를 미리 목록화하면 다음 단계 작업이 빠르게 시작됨.
